## Overview

**Linux CMP** is a custom Arch Linux-based distribution engineered from the ground up
for competitive and high-performance gaming. It ships with a fully optimized kernel,
automatic GPU driver configuration, and a complete gaming stack — ready out of the box.

### Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| **Base** | Arch Linux | Rolling release, latest Mesa/drivers/kernel, AUR access, SteamOS lineage |
| **Kernel** | linux-cachyos | BORE scheduler, fsync/futex2 patches, x86-64-v3 optimizations |
| **Desktop** | KDE Plasma 6 | Wayland-first with HDR/VRR, X11 fallback, best gaming compositor (gamescope integration) |
| **GPU Strategy** | Universal OOTB | NVIDIA-dkms + Mesa RADV + ANV all pre-installed, auto-configured at first boot |
| **Package Source** | CachyOS repos + Chaotic-AUR | Pre-built optimized packages without manual AUR compilation |

## Build Instructions

### Prerequisites

Build this ISO **on an existing Arch Linux system** (or an Arch VM/container):

```bash
# Install archiso and dependencies
sudo pacman -Sy --needed archiso git base-devel

# Clone this repository
git clone <repo-url> linux_cmp && cd linux_cmp

# Build the ISO (requires root)
sudo ./build.sh build
```

The ISO will be created in `./out/`.

### Build Commands

| Command | Description |
|---------|-------------|
| `sudo ./build.sh build` | Full build (install deps → prepare profile → generate ISO) |
| `sudo ./build.sh clean` | Remove work directory |
| `sudo ./build.sh rebuild` | Clean + full build |

### Build Requirements

- **OS**: Arch Linux (or Arch-based)
- **RAM**: 8GB minimum (16GB recommended for faster squashfs)
- **Disk**: ~25GB free for work directory
- **Packages**: `archiso`, `squashfs-tools`, `libisoburn`, `mtools`, `dosfstools`
- **Internet**: Required to download packages during build

## Architecture Details

### FASE 1: Boot & GPU Detection Flow

```
BIOS/UEFI → GRUB (gaming cmdline) → linux-cachyos kernel (BORE scheduler)
    → systemd initramfs (KMS early) → gpu-driver-setup.service (first boot only)
        → lspci GPU detection
        → NVIDIA? → nvidia-dkms + modprobe opts + Wayland env
        → AMD?    → Mesa/RADV + amdgpu modprobe opts
        → Intel?  → Mesa/ANV + i915/xe opts
        → Hybrid? → PRIME render offload configuration
        → mkinitcpio rebuild with GPU modules
    → sddm (Wayland) → KDE Plasma 6 Wayland session
    → linux-cmp-welcome.sh (user groups, Proton-GE symlinks)
```

### FASE 2: Gaming Optimization Stack

| Layer | Component | Optimization |
|-------|-----------|-------------|
| **Kernel** | linux-cachyos | BORE scheduler, fsync/futex2, x86-64-v3, `preempt=full` |
| **Kernel cmdline** | GRUB | `mitigations=off`, `tsc=reliable`, `pcie_aspm=off`, `processor.max_cstate=1` |
| **Scheduler** | sysctl | 500μs granularity, autogroup, low migration cost |
| **Memory** | sysctl | `swappiness=1`, `max_map_count=2B`, `overcommit=1`, THP enabled |
| **Network** | sysctl + BBR | TCP Fast Open, BBR congestion, 16MB buffers |
| **Storage** | udev | kyber/NVMe, mq-deadline/SSD, bfq/HDD per-device |
| **CPU** | udev + GRUB | Performance governor, max turbo, C1 max C-state |
| **GPU** | modprobe | DRM KMS, PAT, preserve VRAM on suspend |
| **Audio** | PipeWire | 256 quantum @ 48kHz = 5.3ms latency |
| **Process** | limits.d | RT priority 98, `nice -20`, unlimited memlock |
| **Wine/Proton** | env vars | `WINEFSYNC=1`, `WINEESYNC=1`, FSR enabled |
| **Compositor** | Gamescope | Per-game Wayland, VRR/HDR, FSR upscaling |

### FASE 3: Security Considerations

> ⚠️ **WARNING**: `mitigations=off` in the GRUB cmdline disables Spectre/Meltdown
> mitigations for ~5-15% performance gain. This is appropriate for a **dedicated gaming
> machine** but **NOT suitable for a server or multi-user system**. Remove this parameter
> if using this distro in any other context.

## Post-Build: Testing the ISO

```bash
# Quick test with QEMU (UEFI, 8GB RAM, GPU passthrough optional)
qemu-system-x86_64 \
    -enable-kvm \
    -m 8G \
    -cpu host \
    -smp 4 \
    -bios /usr/share/ovmf/OVMF.fd \
    -drive file=out/linux_cmp-*.iso,format=raw,media=cdrom \
    -boot d \
    -vga virtio \
    -display gtk,gl=on

# Write to USB (DESTRUCTIVE - verify device!)
sudo dd if=out/linux_cmp-*.iso of=/dev/sdX bs=4M status=progress oflag=sync
```

## Customization

### Adding Packages
Edit `archiso-profile/packages.x86_64` — one package per line, comments with `#`.

### Changing Kernel Parameters
Edit `archiso-profile/airootfs/etc/default/grub` → `GRUB_CMDLINE_LINUX_DEFAULT`.

### Switching to X11 Default
Edit `archiso-profile/airootfs/usr/local/bin/linux-cmp-postinstall.sh`, change SDDM config:
```ini
[General]
DisplayServer=x11
```

### Disabling Mitigations Bypass
Remove `mitigations=off` from GRUB cmdline in `etc/default/grub`.
